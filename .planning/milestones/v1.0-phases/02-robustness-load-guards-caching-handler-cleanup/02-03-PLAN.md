<objective>
Clean up settings handlers a dapp registered via `onChange()`/`onAnyChange()` when that dapp
is disabled via `disableDapp()` (ROB-04). The settings plugin subscribes to the existing
`dx:dapp:disabled` shell event in `init()` and prunes only the disabled dapp's own handlers —
leaving the `_shell` toggle-bridge intact so re-enable still works. Fully event-driven: no new
shell API, no shell→plugin coupling, no duck-typing (D-13).

Purpose: a disabled dapp stops leaking settings handlers and can no longer receive setting
changes it should no longer see.
Output: `plugins/settings/src/index.ts` gains a `cleanup(dappId)` helper + subscribe/unsubscribe
lifecycle; `plugins/settings/tests/settings.test.ts` gains regression coverage.
</objective>
