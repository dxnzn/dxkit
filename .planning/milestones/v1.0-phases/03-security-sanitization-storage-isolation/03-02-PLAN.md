SEC-02 + folded wallet correctness fixes: make the wallet plugin's localStorage key configurable so multiple DxKit apps on one origin don't collide, and fix two data-integrity defects (empty-accounts and silent reconnect failure) in the same single file touch.

Purpose: Two DxKit apps on a shared IPFS-gateway origin must persist wallet-provider selection independently; and the wallet must never emit a malformed connected-state or silently swallow a reconnect failure — consistent with the milestone's "failures are visible" value.
Output: `WalletOptions.storageKey`, the closure-variable swap, WR-02 empty-accounts throw + address-assertion removal, WR-03 reconnect `dx:error`, and the regression/isolation tests.

## Trust Boundaries

|----------|-------------|
| shared-origin localStorage → per-app wallet state | Multiple DxKit apps on one origin share the localStorage namespace; the storage key is the isolation boundary (ASVS V3 partial — provider-selection persistence). |
| injected EIP-1193 provider → wallet state/events | The provider's eth_requestAccounts response is untrusted input shaping emitted wallet state. |
