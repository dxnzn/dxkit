<objective>
SEC-01: Add an optional template sanitizer hook to the lifecycle manager that runs on fetched template HTML immediately before `innerHTML` injection, fail-closed, without introducing any bundled sanitizer.

Purpose: Give consumers a bring-your-own (DOMPurify or equivalent) seam to neutralize XSS in fetched template HTML before it hits the DOM, while keeping the unconfigured default byte-for-byte identical to 0.1.5.
Output: `TemplateSanitizer` type + `sanitizeTemplate` option on `LifecycleManagerOptions`, the slotted mount-time call with a distinct `:sanitize` error source, the barrel export, and the sanitizer test coverage.
</objective>

| framework consumer config → lifecycle manager | The sanitizeTemplate function is consumer-supplied code executed in the mount flow. |
