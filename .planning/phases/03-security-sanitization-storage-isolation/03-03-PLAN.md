<objective>
D-04/D-05 breaking change: restructure ShellConfig's flat lifecycle-loader passthrough into a nested `lifecycle?: LifecycleManagerOptions` group — the seam the SEC-01 sanitizer needs to reach shell consumers — and hard-fail (throw) on the removed flat shape for untyped consumers. This also closes the Phase 2 gap where timeout/cacheTemplates were unreachable from createShell().

Purpose: SEC-01's sanitizer (and the Phase 2 timeout/cache knobs) must be configurable through createShell(); grouping related config and hard-deprecating the flat loaders is the owner's explicit, justified alpha-window breaking change.
Output: the nested ShellConfig.lifecycle type, the createShell runtime guard + forwarding, and a fully migrated shell test suite proving passthrough and the runtime throw.

This is a BREAKING CHANGE — the commit MUST carry a `BREAKING CHANGE:` footer with migration notes (flat `scriptLoader`/`styleLoader`/`templateLoader` → `lifecycle: { scriptLoader, styleLoader, templateLoader }`).
</objective>
