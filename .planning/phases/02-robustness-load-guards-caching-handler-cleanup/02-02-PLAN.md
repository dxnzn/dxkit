<objective>
Hoist the per-call manifest sort out of `Router.resolve()` into the `createRouter` closure
(ROB-02, D-08) so `resolve()` stops re-sorting the manifest list on every navigation. The
router is immutable and fully rebuilt whenever manifests change (`shell.rebuildRouter()` on
enable/disable), so a construction-time sort satisfies "re-sort only when manifests change."

Purpose: eliminate redundant O(n log n) work on the hot navigation path with zero public
API change.
Output: `src/router.ts` sorts once at construction; `tests/router.test.ts` gains a regression test.
</objective>
