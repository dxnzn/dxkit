<objective>
Surface two previously-silent mount failures on the core shell/lifecycle path:
- DIAG-01: emit `dx:error` (source `shell:mount`) when the shell can't resolve `#dx-mount`, replacing the silent no-op at src/shell.ts:288-289.
- DIAG-03: clear the mount container (`container.innerHTML = ''`) on post-injection load failure (entry-script AND dependency-script) so a failed mount never leaves stale dapp DOM visible.

Purpose: A developer whose dapp fails to mount currently sees nothing — no event, stale DOM, and currentDapp stuck at null. This makes both conditions observable via the existing `dx:error` event without reshaping its payload.

Output: Modified src/shell.ts and src/lifecycle.ts plus regression tests in tests/shell.test.ts and tests/lifecycle.test.ts.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| shell/lifecycle → `dx:error` listeners | Failure messages emitted on the bus are visible to any registered `dx:error` handler (dapps, plugins, dev tooling). |
| dapp DOM ← lifecycle | `container.innerHTML` is written/cleared by the shell; a failed mount must not leave a prior dapp's DOM addressable. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01A-01 | Information Disclosure | `shell:mount` / `lifecycle:<id>` error messages | low | accept | Messages include only `manifest.id` (developer-authored, already public in manifests/nav) and the DOM-not-found condition — no user data or secrets are in scope on this path. |
| T-01A-02 | Denial of Service | mountDapp missing-container branch | low | accept | Emit-every-time (D-04) with no dedupe can repeat, but each emit is a bounded synchronous CustomEvent; the repetition is itself the intended signal (consistent with D-09). Dedupe explicitly rejected/deferred. |
| T-01A-03 | Tampering | stale dapp DOM after failed mount | low | mitigate | `container.innerHTML = ''` on post-injection failure removes any partially-injected/prior-dapp DOM so a failed mount leaves no addressable stale nodes (DIAG-03). |

No high-severity threats on this path (block-on threshold is `high`). No package installs in this phase (zero-dep posture), so no supply-chain/legitimacy checkpoint applies.
