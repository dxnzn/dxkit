<objective>
Fix the mount race (D-01/D-02/D-03/D-04) so the shell honors last-navigation-wins instead of last-finisher-wins, and land the dedicated stress suite that proves it (TEST-01).

Purpose: Under rapid AтЖТBтЖТA navigation with slow loaders, the current code lets whichever mount *finishes* last win the DOM and currentDappId (root cause: `lifecycle.mount()` unmounts the previous dapp at the top but commits `currentDappId` only after all awaits at `src/lifecycle.ts:364`, and `shell.ts`'s `pendingMountId` only dedupes same-dapp mounts). This violates the mount/unmount pairing dapps rely on for teardown. This is a straight correctness bug fixed in-phase alongside its regression suite.

Output: A generation-guard fix in `src/lifecycle.ts` + `src/shell.ts`, generation-guard unit tests in `tests/lifecycle.test.ts`, and a new `tests/stress.test.ts` covering the full D-03 race matrix тАФ suite ships green (no `test.fails`).
</objective>
